## Critical findings (remediation today)

### CF-1 — CVC in plaintext over HTTP body (Req 3 + 4)
`services/batch-processor/src/processor.ts:217` passes `record.cvc` into the activation `registerCard` body. HMAC signs the hash but does not encrypt the body. On the internal ALB that's HTTP, so CVC is readable to any party with wire-level visibility inside the VPC.

**Fix:** remove CVC from the batch register payload. Two options:
1. Drop CVC entirely — cardholder supplies it in the mobile app post-tap, Vera vault stores it.
2. If it must flow from batch, POST it to a dedicated encrypted endpoint (`POST /api/cvc`) that wraps the value in a one-shot envelope.

### CF-2 — Inter-service HTTP on internal ALB (Req 1 + 4)
Every service-to-service call (activation → data-prep, rca → activation, batch-processor → activation) uses `http://internal-vera-internal-…elb.amazonaws.com:<port>/…` in the task defs. HMAC gives integrity but no confidentiality. SAD blobs, ParamBundle plaintexts, and EMV keys transit those ALB hops in memory on port 3006/3007 in ECS.

**Fix:** migrate internal ALB listeners to TLS (ACM cert for `*.internal.karta.cards`), update task def URLs `http://…` → `https://…`.

### CF-3 — No centralized logging + no alerting (Req 10)
Each service logs to its own CloudWatch group; 30-day retention; no SIEM aggregation; no CloudWatch alarms on error rate / auth failures / unusual inter-service traffic.

**Fix:**
- Raise all `/ecs/palisade-*` log groups to 90-day retention.
- Stream to Security Hub or a SIEM of choice.
- Baseline alarms: error rate > 5 % for 5 min, auth-fail rate > 10/min, rca WS disconnect spike.

### CF-4 — No annual third-party pen test (Req 6.3.2 / 11.3)
PCI DSS 4.0.1 requires an annual external pen test. No evidence of one scheduled or contracted.

**Fix:** contract a QSA or pen-test firm; scope = all public endpoints + card personalisation flow + attestation PKI chain.

## Requirement-by-requirement

### 1. Network perimeter
- **Applicable.**
- **Current:** Public ALB (HTTPS terminated) fronts tap/activation/rca/admin. Internal ALB carries inter-service HTTP on ports 3006/3007 etc. ECS security group `sg-086e7b16e5351f155`. No WAF / Shield observed in code.
- **Gaps:** inter-service HTTP (see CF-2); no AWS WAF rules documented.
- **Remediation:** ACM cert on internal ALB + listener flip to HTTPS; attach AWS WAF rate-limiting rules to public ALB.

### 2. Default-deny + explicit-allow
- **Applicable.**
- **Current:** Per-service ECS task roles (`vera-tap-task`, `vera-data-prep-task`, …). Public ALB routes by host/path.
- **Gaps:** no per-endpoint RBAC on admin UI — every user in the `admin` Cognito group sees every FI's programs / embossing templates / audit log. CORS fallback origin list is broad (`http://localhost:5174,5176`).
- **Remediation:** program-scoped admin role (Cognito custom attribute or Postgres `AdminRole` table); narrow CORS origins to prod hostnames only.

### 3. Protect stored cardholder data
- **Fully applicable.**
- **Current:**
  - **No full PAN persisted** — PAN is tokenised via Vera vault at registration, only `vaultToken` survives in `Card.vaultToken`.
  - **UID:** AES-256-GCM via KMS; plaintext never touches DB.
  - **SAD:** KMS envelope per `SadRecord.sadEncrypted`.
  - **ParamBundle:** same, plus per-field envelope columns (`mkAcEncrypted`, `mkSmiEncrypted`, `mkSmcEncrypted`, `iccRsaPrivEncrypted`) when `PARAMS_PER_COLUMN=1`.
  - **Attestation priv scalar:** never persisted server-side; minted in memory, sent directly to chip via STORE_ATTESTATION, scrubbed.
  - **SCP03 session state:** purged after op completion or 30-min idle (`retention/src/purge.ts`).
- **Gaps:**
  - **CF-1** — CVC in plaintext HTTP body from batch processor.
  - `Card.panLast4`, `Card.cardholderName`, `Card.panExpiryMonth/Year` stored plaintext (RDS-at-rest encrypted, no app-level envelope).
  - `aws-setup.sh` hard-codes GP test key (`40..4F`) as dev default. Production gate is `CARD_OPS_USE_TEST_KEYS` + a NODE_ENV=production prodGuard in `services/card-ops/src/env.ts` — verify the gate works end-to-end.
- **Remediation:**
  - Remove `cvc` from batch processor payload (CF-1).
  - Add row-level encryption for metadata (`panLast4`, `cardholderName`, expiry) with KMS envelope.
  - Ensure `CARD_OPS_USE_TEST_KEYS` check runs in CI assertion.

### 4. Encrypt transmission
- **Fully applicable.**
- **Current:** ALB TLS termination for public traffic. Mobile WS over `wss://`. HSTS via `helmet()` (default config).
- **Gaps:** CF-2 (inter-service HTTP).
- **Remediation:** see CF-2.

### 5. Malware protection
- **Applicable.**
- **Current:** ECR image scanning enabled; `npm audit --audit-level=high` in CI.
- **Gaps:** `continue-on-error: true` on the audit step — vulns slip through silently.
- **Remediation:** remove `continue-on-error`; add Dependabot for security-only auto-PRs; generate SBOM per CI build.

### 6. Secure SDLC
- **Fully applicable.**
- **Current:** TypeScript strict; Zod schemas on every route; Prisma exclusively (no raw SQL — audited by `grep '\\\$queryRaw'`: **zero hits**); vitest suites (~835 tests); no explicit SAST.
- **Gaps:** no SAST plugin (`eslint-plugin-security`); no documented threat model; no regular security testing cadence.
- **Remediation:** add ESLint security plugin + PII-log pre-commit grep; write `THREAT_MODEL.md`; schedule annual pen test.

### 7. Restrict access (need-to-know)
- **Applicable.**
- **Current:** Cognito `admin` group gates the admin UI; HMAC `keyId`-scoped service-to-service auth; per-service task roles.
- **Gaps:** admin users all see all FIs; no admin action audit (APDU traffic is logged, UI actions are not); HMAC key rotation cadence undefined.
- **Remediation:** program-scoped RBAC; wrap admin routes with an audit middleware that writes to a new `AdminAuditLog` table; document quarterly HMAC rotation.

### 8. Authentication
- **Fully applicable.**
- **Current:** Cognito JWT + MFA (pool-level); WebAuthn for cardholder; HMAC-SHA256 with ±60 s replay window for S2S; WS upgrade token (HMAC-signed, 60 s TTL).
- **Gaps:** cardholder action audit missing; HMAC rotation undocumented; Cognito MFA claim is only documented in a comment, not asserted at startup.
- **Remediation:** per-request audit middleware on cardholder routes; `describeUserPool` assertion at boot.

### 9. Physical access
- **N/A** — cloud-hosted on AWS.

### 10. Log + monitor
- **Fully applicable.**
- **Current:** CloudWatch per-service log groups (30-day retention); APDU audit log (`card-ops/src/ws/apdu-audit.ts`) with redaction for STORE DATA / PUT KEY / SCP03-enciphered APDUs.
- **Gaps:** CF-3 — no SIEM aggregation; no alarms.
- **Remediation:** see CF-3. Add Security Hub, GuardDuty.

### 11. Test security regularly
- **Applicable.**
- **Current:** npm audit (soft), ECR scan on push, vitest unit suite.
- **Gaps:** CF-4 — no pen test scheduled; no DAST; no fuzzing for parsers.
- **Remediation:** see CF-4; add OWASP ZAP passive scan to CI; fuzz the embossing-file parsers (`batch-processor/src/parsers/*.ts`).

### 12. Security policy
- **Applicable.**
- **Current:** inline code comments, `PROTOTYPE_PLAN.md` architectural doc. No `SECURITY.md` / `INCIDENT_RESPONSE.md` / `POLICY.md`.
- **Gaps:** no written policy covering data classification, encryption standards, incident response, key rotation, change management.
- **Remediation:** write `SECURITY.md`, `INCIDENT_RESPONSE.md`, `.well-known/security.txt`.
